JavaScript for Dummies!

In this sprint we began learning JavaScript. This language enables our websites to become interactive! In the first few weeks of foundations we learnt that HTML defines what goes on a web page, and css makes it look pretty. With Javascript we’re able to define processes that add functionality. A good analogy could be a film crew where HTML is the writer, CSS is the wardrobe/set designer and JavaScript is the director.

Control Flow & Loops

Control flow is the order in which a computer carries out defined processes. By using loops, we can check through a set of information. By utilizing these features we can modify and assess varying input data.

Here’s a day to day example of how control flow and looping works. For simplicity, the inner workings of the washDish function are not yet defined.

      
let dishes = [‘pot’, ‘pan’, ‘plate’, ‘big plate’, ‘ramekin’, tiny bowl’]	//this is an array of dishes

washDish(dish){	/*here’s our process for washing a dish!*/
  scrub(dish)
  rinse(dish)	//we haven’t yet added functions for scrub() and rinse()
  addtoDryingRack(dish)	//maybe this one could take said dish out of the dishes array and add to a new array full of items in the drying rack!
}

doTheDishes(setOfDishes){		//our process for doing a set of dishes!
  For each(dish in setOfDishes){	//this loop says for each dish we will do the washDish function!
    washDish(dish)
  }
}

doTheDishes(dishes) //here we are calling the function. This says, “you now know how to do the dishes so go and get started!”

When we do the dishes, we take each dish; wash, rinse, and add to the drying rack, before doing the next dish. This is what is happening here!

What is the DOM?

The document object model is the structure that defines our web pages. It is organised hierarchically so that parents can define how children behave but not the other way around. Referring back to the analogy about a film crew, if HTML is the writer and the parent is the script, the actor (child) can’t say “Hey I think we should change the script”, but the director (JavaScript) can! JavaScript gives us the ability to manipulate the DOM.

The difference between accessing data from arrays and objects.

Arrays are essentially lists of data i.e. a list of characters in the film:

        const characters = [‘Kate’, ‘Eric’, ‘Joaquin’]

We access data at points in the array with an index - characters[1] would return ‘Eric’


Objects can contain more information about the object 

        
const character = {
  name: 'Kate',
  age: '32',
  occupation: 'Dish Washer',
}

To access the value of age in this character object we can use character.age, which gives ‘32’


You could store this information in an array i.e.

        
const character = [‘Kate’, ‘32’, ‘Dishwasher’]

But due to the nature of the information, and for ease of future programming, it’s best to use an object in this case!

What are functions and why are they helpful?

Functions carry out tasks that are likely to be used multiple times. For instance imagine having to tell someone how to wash a dish every time they finished another. Functions give the code the ability to recall the task without it having to be typed out again and again.
